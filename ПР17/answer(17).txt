1. В чем разница между управляемыми и неуправляемыми компонентами в формах React?
Управляемые компоненты хранят значение поля в состоянии React (state) и обновляются через колбэк (например, onChange). Неуправляемые компоненты получают значения напрямую из DOM (через ref).

2. Когда следует использовать управляемые компоненты, а когда - неуправляемые?
Управляемые компоненты используются когда нужен немедленный доступ к данным, валидация или управление значением на каждом изменении. Неуправляемые — для простых форм, когда важно быстродействие и не нужна промежуточная обработка.

3. Какие методы используются для обработки событий в формах React?
Основные методы: onChange (для обработки изменений в полях ввода), onSubmit (для отправки формы). События обрабатываются аналогично нативным, но с использованием camelCase и синтетической системы событий React.

4. Как обрабатывать несколько полей ввода с одним обработчиком?
Для этого используется один обработчик события onChange, который определяет, какое поле изменилось, по атрибуту name или другому идентификатору, и обновляет соответствующее значение в состоянии.

5. Какие подходы к валидации форм в React вы знаете?
Основные подходы: валидация на стороне клиента при изменении поля (onChange), при отправке формы (onSubmit), использование HTML5 атрибутов валидации и специализированных библиотек (например, Formik, React Hook Form).

6. Как предотвратить стандартное поведение формы (перезагрузку страницы)?
Необходимо вызвать метод preventDefault() у объекта события (event) в обработчике onSubmit формы. Это предотвратит стандартную перезагрузку страницы при отправке формы.

7. Что такое "поднятие состояния" (lifting state up) в контексте форм?
Это подъем состояния формы из дочернего компонента в родительский компонент. Используется, когда данные формы нужны нескольким компонентам или когда родительский компонент должен управлять состоянием формы и координировать взаимодействие между компонентами.
