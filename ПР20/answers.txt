Ответы на вопросы для самоконтроля
1. В чем преимущества использования Express.js по сравнению с чистым Node.js для создания серверов?
Express.js упрощает разработку серверов за счет:

Декларативного роутинга: легко определять обработчики для URL и HTTP-методов.
Middleware-архитектуры: возможность подключать готовые или кастомные middleware (логирование, аутентификация, парсинг тела запроса).
Встроенных инструментов: автоматическая обработка статических файлов, шаблонизаторов (Pug, EJS), CORS.
Упрощенной работы с запросами и ответами: удобные методы для отправки JSON, файлов, редиректов.
Сообщества и экосистемы: огромное количество готовых пакетов для расширения функционала.
2. Что такое middleware в Express.js и каков порядок их выполнения?
Middleware — это функции, обрабатывающие запросы до попадания в конечные обработчики. Они могут:

Модифицировать объекты req и res.
Завершать запрос (отправлять ответ) или передавать управление следующему middleware через next().
Порядок выполнения:
Middleware вызываются в том порядке, в котором они зарегистрированы в приложении. Например, middleware для логирования должен идти первым, а обработчик ошибок — последним.
3. Как организовать модульную структуру роутеров в Express приложении?
Для модульности используются:

express.Router(): создание отдельных роутеров для разных частей приложения (например, /users, /products).
Разделение файлов: каждый роутер выносится в отдельный файл.
Подключение через app.use(): роутеры монтируются в основное приложение с префиксами (например, app.use('/api/users', usersRouter)).
Это упрощает поддержку кода и изоляцию логики.
4. Какие методы роутинга поддерживает Express.js и для чего они используются?
Основные методы:

app.get(): обработка GET-запросов для получения данных.
app.post(): создание ресурсов через POST-запросы.
app.put()/app.patch(): обновление ресурсов.
app.delete(): удаление ресурсов.
app.all(): обработка любого HTTP-метода для указанного пути.
app.use(): применение middleware ко всем методам или указанному пути.
5. Как обрабатывать параметры URL и query-параметры в Express?
Параметры URL (например, /users/:id): доступны через req.params.id в обработчике.
Query-параметры (например, /search?query=test): извлекаются из req.query в виде объекта.
Express автоматически парсит их, упрощая работу с динамическими маршрутами и фильтрами.
6. В чем разница между app.use() и app.METHOD()?
app.use():
Применяется ко всем HTTP-методам или указанному пути (без привязки к методу).
Используется для middleware (например, парсинг JSON, статические файлы).
app.METHOD() (например, app.get()):
Привязан к конкретному HTTP-методу (GET, POST и т.д.).
Обрабатывает запросы только для указанного метода и пути.
7. Как организовать централизованную обработку ошибок в Express приложении?
Для централизованной обработки:

Создается специальное middleware с четырьмя параметрами (err, req, res, next).
Оно регистрируется после всех роутов и middleware.
Внутри middleware анализируется ошибка, логируется, и клиенту отправляется стандартизированный ответ (например, { error: 'Сообщение' }).
Это позволяет избежать дублирования кода обработки ошибок.
8. Что такое router.param() и для чего он используется?
router.param() — метод для предварительной обработки параметров маршрутов. Например:

Валидация параметра (проверка ID на соответствие формату).
Загрузка данных из базы по параметру (например, поиск пользователя по :userId).
Функция, переданная в router.param(), вызывается один раз при первом обращении к параметру, что оптимизирует повторные запросы.
