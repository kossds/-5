Ответы на вопросы для самоконтроля

1. В чем разница между функциональными и классовыми компонентами в React? Когда следует использовать каждый тип?

Различия:
- Синтаксис: Функциональные компоненты - это функции, классовые - классы ES6
- Состояние: Раньше состояние было только в классовых компонентах, теперь с хуками доступно в функциональных
- Жизненный цикл: Классовые используют методы жизненного цикла, функциональные - хуки useEffect
- this: В классовых компонентах нужно работать с this, в функциональных - нет

Когда использовать:
- Функциональные компоненты: Новые проекты, простые компоненты, с хуками - любые компоненты
- Классовые компоненты: Легаси код, когда нужны Error Boundaries

2. Что такое props и state? Как они отличаются и какие правила их использования?

Props:
- Передаются от родителя к потомку
- Неизменяемые (read-only)
- Используются для передачи данных и callback-функций

State:
- Внутреннее состояние компонента
- Изменяемое с помощью setState (классовые) или useState (функциональные)
- Инициализируется в компоненте

Правила:
- Не изменяйте props напрямую
- Обновляйте state только через специальные методы
- State должен быть минимально необходимым

3. Как работает поднятие состояния (lifting state up) и в каких ситуациях это необходимо?

Поднятие состояния - это перемещение состояния в ближайшего общего родителя компонентов, которые нуждаются в общих данных.

Ситуации использования:
- Когда несколько компонентов должны отражать одни и те же данные
- Для синхронизации состояния между родственными компонентами
- Когда дочерние компоненты должны общаться друг с другом

4. Какие методы жизненного цикла классовых компонентов вы знаете и для чего они используются?

Монтирование:
- constructor() - инициализация состояния
- render() - рендеринг компонента
- componentDidMount() - после монтирования, для side effects

Обновление:
- shouldComponentUpdate() - оптимизация рендеринга
- render() - рендеринг обновлений
- componentDidUpdate() - после обновления

Размонтирование:
- componentWillUnmount() - очистка ресурсов

5. Как работают хуки useState и useEffect? Какие правила использования хуков существуют?

useState:
- Возвращает массив [значение, функция-сеттер]
- Используется для управления состоянием

useEffect:
- Выполняет side effects
- Принимает функцию и массив зависимостей
- Может возвращать функцию очистки

Правила хуков:
- Вызывайте хуки только на верхнем уровне
- Вызывайте хуки только из React-функций

6. Что такое ключи (keys) в React и почему они важны при рендеринге списков?

Ключи - это специальные строковые атрибуты, которые помогают React идентифицировать элементы в списке.

Важность:
- Помогают React определить, какие элементы изменились
- Улучшают производительность при обновлении списков
- Предотвращают ошибки и непредсказуемое поведение

7. Как обрабатывать события в React и чем это отличается от нативного JavaScript?

Отличия:
- В React события называются в camelCase (onClick vs onclick)
- В React нельзя вернуть false для предотвращения поведения по умолчанию
- В React события - это SyntheticEvent

Обработка:
- Передаем функцию-обработчик как prop
- В классовых компонентах привязываем контекст или используем стрелочные функции

8. Что такое управляемые и неуправляемые компоненты? В чем их разница?

Управляемые компоненты:
- Значение контролируется React через state
- Изменения обрабатываются через обработчики событий
- Рекомендуемый подход

Неуправляемые компоненты:
- Значение хранится в DOM
- Доступ через refs
- Используются для интеграции с не-React кодом

9. Как работает контекст (Context) в React и когда его следует использовать?

Контекст позволяет передавать данные через дерево компонентов без передачи props на каждом уровне.

Использование:
- Глобальные данные (тема, язык, пользователь)
- Когда props нужно передавать через много уровней
- Альтернатива Redux для простых случаев

10. Какие преимущества предоставляют кастомные хуки и как создавать собственные хуки?

Преимущества:
- Переиспользование логики состояния
- Разделение сложной логики на простые функции
- Улучшение читаемости и тестируемости

Создание:
- Функция, начинающаяся с "use"
- Может использовать другие хуки
- Возвращает значения, которые нужно использовать в компонентах
