1. Что такое Node.js и в чем его преимущества для создания серверов?
Node.js — это серверная среда выполнения JavaScript, работающая на движке V8. Её ключевая особенность — асинхронная неблокирующая архитектура, что позволяет эффективно обрабатывать множество одновременных запросов. Преимущества включают:

Высокую производительность для I/O-интенсивных задач (например, обработка файлов, сетевых запросов).
Единый язык (JavaScript) для клиентской и серверной разработки.
Доступ к обширной экосистеме пакетов через npm.
Легковесность и гибкость по сравнению с традиционными серверными платформами.
2. Как создать простой HTTP-сервер в Node.js?
Для создания сервера используется встроенный модуль http. Основные шаги:

Импортировать модуль http.
Создать сервер с помощью метода createServer, определив обработчик запросов и ответов.
Настроить прослушивание порта с помощью метода listen.
Обработчик получает объекты запроса (req) и ответа (res), через которые управляются заголовки, статусы и содержимое ответа.
3. В чем разница между методами GET, POST, PUT и DELETE?
GET: Запрашивает данные без изменения состояния сервера. Параметры передаются в URL.
POST: Отправляет данные для создания нового ресурса. Может быть неидемпотентным (повторные запросы создают дубликаты).
PUT: Обновляет существующий ресурс или создаёт его с указанным идентификатором. Идемпотентен (результат одинаков при повторных запросах).
DELETE: Удаляет ресурс по указанному идентификатору. Также идемпотентен.
4. Что такое middleware и для чего оно используется?
Middleware — это промежуточные функции, обрабатывающие запросы до попадания в основной обработчик. Они позволяют:

Модифицировать объекты запроса и ответа.
Выполнять предварительные проверки (например, аутентификацию).
Логировать запросы или обрабатывать ошибки.
Передавать управление следующему middleware с помощью функции next().
Примеры: парсинг JSON-данных, проверка прав доступа, сжатие ответов.
5. Как обрабатывать различные HTTP-методы для одного URL?
В обработчике запроса проверяется свойство req.method, чтобы определить тип метода (GET, POST и т.д.). В зависимости от значения запускается соответствующая логика. Во фреймворках (например, Express.js) для этого используются специальные методы (app.get(), app.post()), которые автоматически фильтруют запросы по методу.

6. Что такое CORS и зачем нужны соответствующие заголовки?
CORS (Cross-Origin Resource Sharing) — механизм безопасности браузера, ограничивающий кросс-доменные запросы. Чтобы разрешить такие запросы, сервер должен отправлять заголовки:

Access-Control-Allow-Origin: указывает разрешённые домены.
Access-Control-Allow-Methods: перечисляет допустимые HTTP-методы.
Access-Control-Allow-Headers: определяет разрешённые заголовки в запросе.
Без этих заголовков браузер блокирует ответы от сервера, если домен запроса не совпадает с доменом сервера.
7. Как организовать чтение тела POST-запроса в Node.js?
В «чистом» Node.js тело запроса читается через события потока данных:

Событие data собирает фрагменты данных из запроса.
Событие end сигнализирует о завершении передачи, после чего данные парсятся (например, из JSON).
Во фреймворках (например, Express.js) для этого используются middleware (например, express.json()), которые автоматически парсят тело запроса и сохраняют результат в объекте req.body.
